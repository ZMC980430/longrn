# 示例 SDD：Longrn — 终生学习者智能学习路径系统

文档元信息
- 标题：Longrn SDD
- 版本：0.1
- 作者：项目团队
- 日期：
- 状态：草稿

1. 概述
Longrn 是一个面向终生学习者的智能学习路径系统，深度集成 Obsidian 与 Logseq，用知识图谱与语义搜索为用户生成个性化学习路径，并自动生成结构化笔记与双链。

2. 目标与成功标准
- 自动从用户笔记库构建知识图谱并生成语义索引。
- 基于用户目标与现有知识状态生成可执行的学习路径（至少提供快速/深入两种路径）。
- 自动生成笔记并建立内部双链，提升笔记互联质量。
- 验收：在一个测试 Vault 上能自动生成并链接 3 个连续主题笔记，CI 通过单元/集成测试。

3. 主要功能需求
- FR-1：扫描并解析本地 Vault（Obsidian）或 Graph（Logseq），提取标题、标签与双链。
- FR-2：构建个人知识图谱与语义向量索引。
- FR-3：接受学习目标并生成一条或多条学习路径（可导出为 Canvas）。
- FR-4：根据路径批量生成结构化 Markdown 笔记并自动建立链接。

4. 系统架构（高层）
- 客户端插件：Obsidian 插件、Logseq 插件（负责与本地笔记交互）。
- 服务/库：知识库构建引擎、路径规划器、笔记生成器（可本地运行或调用模型服务）。
- 存储：本地 Markdown 文件、可选的向量数据库（Faiss/Chroma）、索引存储。

5. 模块划分
- 知识库构建引擎：文件解析、图谱构建、向量化。
- 路径规划器：领域图谱匹配、路径搜索算法（多路径生成）。
- 智能笔记生成器：模板渲染、自动链接引擎。
- 插件适配层：Obsidian 与 Logseq 的 SDK 封装。

6. 接口示例
- 插件命令：generate-learning-path（触发流程）
- 内部函数：buildKnowledgeBase(): Map<string, Note>
- 导出格式：Markdown 文件夹结构或 Logseq 页面批量创建

7. 数据模型
- Note: id, title, path, content, embeddings, tags
- KnowledgeGraph: nodes, edges, edge_type

8. 部署与运行
- 本地运行：以插件形式运行于 Obsidian/Logseq；或通过 CLI 在用户 Vault 上执行批处理。
- CI：至少包含构建、类型检查与单元测试。

9. 测试计划
- 单元：核心算法与文本处理函数。
- 集成：在一个示例 Vault 上验证路径生成与笔记输出。

10. 风险与缓解
- 风险：插件 API 差异导致适配复杂。缓解：先以文档与示例为主，逐步兼容。
