# SDD 完成度检查清单（Checklist）

在提交实现 PR 前，逐项确认如下内容：

- [ ] 文档元信息（标题/版本/作者/状态）
- [ ] 项目概述与目标清晰
- [ ] 功能需求按编号列明并有验收标准
- [ ] 非功能需求明确（性能/可靠性/安全等）
- [ ] 关键用例与用户流程覆盖主要场景
- [ ] 系统架构图或架构概览
- [ ] 模块划分与接口定义（含示例）
- [ ] 数据模型与示例数据
- [ ] 依赖项与第三方服务说明
- [ ] 部署步骤与回滚策略
- [ ] 测试计划（含必过的测试用例）
- [ ] 监控/告警/运维要点
- [ ] 风险/限制/已知问题列出并评价
- [ ] 评审记录与负责人签名

说明：若有未完成项，请在 SDD 中标注为风险/待办并更新 PR 描述。
